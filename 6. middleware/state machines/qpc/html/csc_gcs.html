Guideline Compliance Summary (GCS)

Guideline Enforcement Plan (GEP)Deviations

As recommended in [MISRA4], this section presents the Guideline Compliance Summary (GCS) (previously called MISRA Compliance Matrix), which lists all MISRA-C:2023 guidelines and indicates the compliance status as well as how the guidelines have been checked.

Compliance Matrix Conventions

The MISRA Compliance Summary is presented in the tabular form:

Guideline PCLP  QP/C   QS  Headline
... ... ... ... ...

Whereas the meaning of the table columns is as follows:

Guideline: lists the MISRA-C:2023 guideline (directive or rule) number followed by the guideline classification in parentheses (as categorized in [MISRA3]):

  • (M) Mandatory — guidelines for which violation is never permitted
  • (R) Required — guidelines which can only be violated when supported by a deviation defining a set of clear restrictions, requirements and precautions
  • (A) Advisory — recommendations to be followed as far as reasonably practical. Violations are identified but are not required to be supported by deviation.

PCLP: lists whether a given MISRA guideline is checked by PCLP (see also Configuring PCLP). This column might contain the following:

  • Yes — the guideline is checked by PCLP
  • Part — the guideline is partially checked by PCLP
  • Assist — PCLP provides some assistance in checking the guideline
  • No — the guideline is NOT checked by PCLP

QP/C: lists the compliance status of the QP Framework code. This column might contain the following:

  • M — means that the QP Framework code complies with the MISRA guideline by Manual inspection
  • A — means that the QP Framework code complies with the MISRA guideline by Automatic inspection (via PCLP)
  • DVP — Link to the Deviation Permit for the QP Framework and QP Application code.
  • DVR — Link to the Deviation Record for the QP Framework code only.
Remarks
A Deviation Permit is provided in this column when a given MISRA guideline violation is imposed on the QP Application by the design of the QP Framework. Typically, such MISRA guideline violation occurs in the QP Framework code as well. In contrast, a Deviation Record is provided when a given MISRA guideline violation occurs only inside the QP Framework source code and has no impact on the QP Application.

QS: lists the compliance status of the QS software tracing instrumentation. This column might contain the following:

  • M — means that the QS instrumentation complies with the MISRA guideline by Manual inspection
  • A — means that the QS instrumentation code complies with the MISRA guideline by Automatic inspection (via PCLP)
  • DVP — Link to the Deviation Permit for the QS instrumentation and QP Application code, when QS tracing is enabled.
  • DVR — Link to the Deviation Record for the QS instrumentation code only.
Remarks
A Deviation Permit is provided in this column when a given MISRA guideline violation is imposed on the QP Application by the QS software tracing instrumentation. Typically, such MISRA guideline violation occurs in the QS tracing implementation code as well. In contrast, a Deviation Record is provided when a given MISRA guideline violation occurs only inside the QS software tracing instrumentation code and has no impact on the QP Application.

Headline: contains the MISRA guideline headline, as published in [MISRA3].

Remarks
The use of the MISRA-C:2023 guideline headlines in this document has been licensed to Quantum Leaps, LLC by MIRA Ltd. (Reference: SB/VH/Con/315/MISRA/Sept99, January 6. 2015)
